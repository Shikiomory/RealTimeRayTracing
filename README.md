# RealTimeRayTracing

Высокопроизводительный движок трассировки лучей, ориентированный на работу в реальном времени.

## Технологии и библиотеки
* **C++17**: Основной язык разработки.
* **SFML 2.6.1**: Используется для управления окном, обработки ввода и вывода финального буфера кадра на экран.
* **OpenMP**: Параллельные вычисления для распределения нагрузки рендеринга на все ядра CPU.

## Ключевые оптимизации

### Архитектура и память
* **BVH (Bounding Volume Hierarchy)**: Реализовано дерево с использованием AABB (Axis-Aligned Bounding Boxes) для ускорения поиска пересечений лучей с объектами.
* **Data-Oriented Design**: Вместо использования тяжелых указателей и shared_ptr внедрена система целочисленных индексов (Material IDs).
* **Управление ресурсами**: Применение std::unique_ptr для владения объектами и std::variant для эффективной обработки различных типов материалов и объектов без динамического полиморфизма.
* **Header-only компоненты**: Критически важные вычислительные узлы вынесены в заголовочные файлы для обеспечения максимального инлайнинга кода компилятором.

### Алгоритмы рендеринга
* **Трехпроходный Pipeline**:
    1. **Base Pass**: Трассировка первичных лучей.
    2. **Adaptive Sampling**: Анализ цветовых различий соседних пикселей и запуск дополнительных лучей в наиболее сложные области сцены.
    3. **Upscaling**: Финальный проход для масштабирования и сглаживания изображения.
* **Накопительный буфер (Static TAA)**: Механизм накопления энергии света во времени, позволяющий полностью устранить шум на статических кадрах.
* **Теневая оптимизация (Any-hit)**: Для теневых лучей используется метод раннего выхода — расчет прерывается при нахождении любого препятствия между точкой и источником света.

## Сборка и запуск
1. Убедитесь, что у вас установлена Visual Studio 2022 (x64).
2. Клонируйте репозиторий и откройте файл `realtime_raytracing.sln`.
3. Выберите конфигурацию **Release** и платформу **x64**.
4. Соберите проект. Все необходимые зависимости (SFML) и DLL-файлы будут автоматически настроены и скопированы в выходную директорию благодаря встроенным событиям сборки.
